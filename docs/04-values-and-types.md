# Значения и типы

Shine динамический по умолчанию, но поддерживает фиксированные аннотации типов.

## Встроенные типы

| Тип | Пример | Описание |
|---|---|---|
| `Int` | `42` | 64-битное целое число |
| `Float` | `3.14` | 64-битное число с плавающей точкой |
| `Number` | `10`, `2.5` | Аннотация, принимающая `Int` и `Float` |
| `String` | `"Hello"` | Строка Unicode |
| `Bool` | `true` | Логическое значение |
| `List` | `[1, 2, 3]` | Основная коллекция |
| `Dictionary` | `{"name": "Amin"}` | Упорядоченные пары ключ — значение |
| `None` | `none` | Отсутствие полезного значения |

Runtime также знает служебные значения `Function` и `Range`. Они могут отображаться через `type`, но не используются как типовые аннотации в MVP.

## Целые числа

```shine
age: Int = 16
population = 8_000_000_000
negative = -42
```

Диапазон `Int` соответствует Rust `i64`: от −9 223 372 036 854 775 808 до 9 223 372 036 854 775 807. Переполнение вызывает `Value Error`, а не необъяснимую аварийную остановку.

## Числа с плавающей точкой

```shine
price: Float = 19.99
speed = 3e8
small = 1.5e-10
```

`Float` соответствует `f64`. Деление `/` всегда возвращает `Float`, даже для двух `Int`.

Специальные значения доступны через `INF` и `NAN`. Сравнения с `NAN` могут не иметь обычного математического порядка.

## Number

`Number` используется только как гибкая числовая аннотация:

```shine
value: Number = 10
value = 2.5
```

Переменная остаётся фиксированно числовой, но может переключаться между `Int` и `Float`.

## Строки

```shine
name: String = "Shine"
message = "Hello, {name}"
```

Строки подробнее описаны в [отдельной главе](11-strings.md).

## Логические значения

```shine
enabled: Bool = true
finished = false
```

`false` и число `0` — разные значения:

```shine
assert(false != 0)
```

## None

```shine
result = none
```

`none` возвращают операции без полезного результата, например `print`, `add`, `clear`, `sort` и `writeFile`.

Аннотация:

```shine
empty: None = none
```

## Списки

Динамический список может содержать значения разных типов:

```shine
mixed = [1, "two", true, none]
```

Типизированный список ограничивает элементы:

```shine
names: List[String] = ["Amin", "Murad"]
values: List[Float] = [1.0, 2.5]
anything: List = [1, "two"]
```

Пустой список допустим для любой аннотации списка:

```shine
measurements: List[Float] = []
measurements.add(1.5)
```

Тип элементов проверяется при создании списка, в `add` и при индексном присваивании. Checker ловит известные несовместимые значения заранее, а runtime проверяет фактические значения. Подробнее: [Списки](10-lists.md).

## Словари

```shine
user = {"name": "Amin", "age": 25}
scores: Dictionary[String, Float] = {"math": 95.5}
```

Словарь сохраняет порядок добавления, поддерживает индексирование по скалярному ключу и проверяет типизированные мутации в checker и runtime. Подробнее: [Словари](23-dictionaries.md).

## Определение типа

```shine
// Print runtime type names.
print(type(10))
print(type(3.14))
print(type("hello"))
print(type([1, 2]))
print(type({"name": "Shine"}))
```

Результаты: `Int`, `Float`, `String`, `List`.

## Совместимость типов

- `Int` принимает только `Int`;
- `Float` принимает только `Float`, без автоматического превращения `Int`;
- `Number` принимает `Int` и `Float`;
- `List[T]` принимает список совместимых элементов;
- динамическая переменная принимает любое поддерживаемое значение.

```shine
radius: Float = 2.0
radius = 3
```

Вторая строка вызывает `Type Error`. Используйте `3.0` или аннотацию `Number`.
