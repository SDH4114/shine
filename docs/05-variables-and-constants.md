# Переменные и константы

## Динамические переменные

Первое присваивание создаёт динамическую переменную:

```shine
value = 10
value = 19.5
value = "Hello"
value = true
```

Её тип может меняться. Повторное присваивание изменяет уже существующую переменную.

## Фиксированный тип

Аннотация после имени закрепляет тип:

```shine
age: Int = 16
age = 17
```

Неверное присваивание:

```shine
age: Int = 16
age = "seventeen"
```

Диагностика объяснит, что `Int` не может получить `String`.

Доступные аннотации:

```shine
count: Int = 10
ratio: Float = 0.5
measurement: Number = 10
title: String = "Report"
ready: Bool = false
items: List = []
names: List[String] = []
empty: None = none
```

Аннотация не делает переменную константой. Значение можно менять, пока новый тип совместим.

## Константы

`const` запрещает повторное присваивание:

```shine
const MAX_USERS = 100
const APP_NAME = "Shine Tool"
```

Также запрещена мутация списка, принадлежащего константе:

```shine
const POINT = [10, 20]
POINT.add(30)
```

Это вызывает `Const Error`.

Для изменяемой копии вызовите `copy()`:

```shine
const ORIGINAL = [1, 2, 3]
working = ORIGINAL.copy()
working.add(4)
```

При создании `const` список получает независимое замороженное содержимое, включая вложенные списки.

## Константы и типы

Константа получает тип из своего единственного значения, поэтому отдельная аннотация запрещена:

```shine
const GRAVITY = 9.80665
const LABELS = ["low", "normal", "high"]
```

Запись `const GRAVITY: Float = 9.80665` выдаёт понятную `Syntax Error`. Если значение должно меняться, используйте один из двух обычных вариантов:

```shine
// Фиксированный тип, значение можно менять.
GRAVITY: Float = 9.80665

// Динамическая переменная.
gravity = 9.80665
```

## Область видимости

Каждый блок функции, условия и цикла создаёт локальную область видимости.

```shine
fn main() {
    result = 10

    if true {
        local = 5
        result = result + local
    }

    print(result)
}
```

`result` существует снаружи и изменяется внутри блока. `local` создаётся внутри `if` и после блока недоступна.

Параметры функции локальны для вызова. Переменная цикла локальна для соответствующей итерации.

## Затенение

Явное объявление с типом или `const` внутри вложенного блока может затенить внешнее имя:

```shine
value: Int = 10

if true {
    value: String = "local"
    print(value)
}

print(value)
```

Внутри блока печатается `local`, после блока — `10`.

## Составные присваивания

Поддерживаются:

```shine
count += 1
count -= 1
count *= 2
count /= 2
```

`/=` использует обычное деление и возвращает `Float`. Поэтому оно несовместимо с переменной, аннотированной как `Int`:

```shine
count: Int = 10
count /= 2
```

Используйте динамическую переменную, `Number`, `Float` или обычное целочисленное выражение с `//`.

## Присваивание элементу списка

```shine
names = ["Amin", "Murad"]
names[1] = "Ali"
```

Индекс должен существовать. Добавление нового элемента за границей списка выполняется через `add`.

## Деструктуризация

Значения списка можно разложить по именам:

```shine
point = [10, 20]
[x, y] = point
```

Количество имён должно точно совпадать с длиной списка.

```shine
fn minMax(values) {
    return [values.min(), values.max()]
}

[minimum, maximum] = minMax([4, 8, 2, 10])
```

Отдельного типа Tuple в MVP нет: для этой задачи используются список, `const` и деструктуризация.

## Рекомендации по именам

- переменные и функции: `camelCase` или `snake_case`;
- константы: `UPPER_CASE`;
- выбирайте имена по смыслу: `averageTemperature`, а не `x`;
- не используйте имена встроенных функций без необходимости.
