# Словари

`Dictionary` хранит пары «ключ — значение» и сохраняет порядок первого добавления ключей.

```shine
user = {
    "name": "Amin"
    "age": 25
}

scores: Dictionary[String, Float] = {
    "math": 95.5
}

user["active"] = true
print(user["name"])
```

## Типы

Динамический `Dictionary` разрешает смешанные значения. Аннотация `Dictionary[Key, Value]` проверяется и статическим checker, и runtime при последующей мутации.

```shine
settings: Dictionary[String, Bool] = {
    "verbose": true
}

settings["safe"] = false
```

Ключом может быть только `String`, `Int`, `Float`, `Bool` или `none`. Численно равные `Int` и `Float` обозначают один ключ: `1` совпадает с `1.0`, а `0.0` с `-0.0`. `NAN` запрещён, потому что он не равен самому себе.

Повтор ключа в литерале создаёт `Key Error`:

```shine
invalid = {1: "first", 1.0: "duplicate"}
```

## Чтение и запись

`dictionary[key]` требует существующий ключ и иначе создаёт `Key Error`. Для необязательного чтения используйте `get`:

```shine
port = config.get("port")
port = config.get("port", 8080)
```

Присваивание новому ключу добавляет его в конец. Присваивание существующему ключу меняет только значение и не перемещает ключ.

## Методы

| Метод | Результат |
|---|---|
| `have(key)` | `Bool`, существует ли ключ |
| `get(key)` | значение или `none` |
| `get(key, default)` | значение или `default` |
| `remove(key)` | `Bool`, был ли ключ удалён |
| `len()` | количество пар |
| `clear()` | удалить все пары |
| `copy()` | независимая копия внешнего контейнера |
| `keys()` | `List` ключей в порядке добавления |
| `values()` | `List` значений в порядке добавления |
| `items()` | `List` двухэлементных списков `[key, value]` |

```shine
loop key in user {
    print("{key}: {user[key]}")
}
```

Цикл по `Dictionary` перебирает ключи. Оператор `key in dictionary` эквивалентен проверке `have(key)`.

## Равенство, const и циклы

Равенство сравнивает пары и не зависит от порядка ключей:

```shine
assert({"a": 1, "b": 2} == {"b": 2, "a": 1})
```

`const Dictionary` создаёт глубоко замороженную копию. Нельзя менять сам словарь и вложенные `Dictionary` или `List`.

Циклические словари можно выводить, сравнивать и замораживать без бесконечной рекурсии. При выводе обратная ссылка обозначается как `{<cycle>}`.

## Производительность и backend

Tree-walking evaluator является эталоном поведения `Dictionary`. Numeric VM не специализирует словари и автоматически передаёт содержащие их функции evaluator. Это сохраняет одинаковую семантику до появления typed MIR/SSA и native backend.
