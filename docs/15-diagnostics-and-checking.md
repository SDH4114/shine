# Ошибки и статическая проверка

Shine старается превращать ошибки в понятные сообщения, а не показывать Rust panic.

## Формат диагностики

```text
Type Error: expected Int, received String

main.shn:4:1
4 | age = "seventeen"
    ^^^^^^^^^^^^^^^^^

A fixed type or operation cannot receive this value.
Suggestion: Use a matching value or remove the type annotation.
```

Сообщение содержит:

1. категорию;
2. краткое описание;
3. имя файла, строку и колонку;
4. исходную строку;
5. указатель на проблемный фрагмент;
6. объяснение;
7. короткую рекомендацию.

Ошибки печатаются в stderr. Процесс завершается кодом `1`.

## Проверка до запуска

```bash
shine check src/main.shn
```

Checker не запускает программу. Поэтому он безопасен для файлов с `input`, `readFile`, `writeFile` и долгими циклами.

Он анализирует:

- lexer и синтаксис;
- ссылки на имена;
- известные типы выражений;
- фиксированные переменные;
- `const` и мутацию списков;
- параметры и результаты функций;
- количество аргументов пользовательских функций;
- управление циклом;
- строковую интерполяцию.

Поскольку язык динамический, некоторые ошибки можно обнаружить только с реальными значениями при `shine run`.

## Категории ошибок

### Syntax Error

Неправильный символ, незакрытая строка или скобка, неполное выражение:

```shine
fn main( {
    print("Hello")
}
```

### Name Error

Неизвестное или повторно объявленное имя:

```shine
print(missingValue)
```

```shine
value: Int = 10
value: Int = 20
```

Для обычного изменения второе присваивание должно быть без аннотации:

```shine
value: Int = 10
value = 20
```

### Type Error

Несовместимый тип переменной, аргумента, результата или операции:

```shine
age: Int = 16
age = "16"
```

### Const Error

Изменение константы или её списка:

```shine
const VALUES = [1, 2]
VALUES.add(3)
```

### Argument Error

Неправильное количество аргументов:

```shine
sqrt(1, 2)
```

### Index Error

Индекс вне строки или списка, либо некорректные границы среза:

```shine
values = [1, 2]
print(values[10])
```

### Key Error

Отсутствующий, повторный или равный `NAN` ключ `Dictionary`:

```shine
user = {"name": "Amin"}
print(user["missing"])
```

Если отсутствие допустимо, используйте `get(key)` или `get(key, default)`.

### Math Error

Деление на ноль или значение вне математического домена:

```shine
result = 10 / 0
```

### Value Error

Значение правильного общего типа, но недопустимого диапазона: нулевой шаг, отрицательное повторение, переполнение, пустой список для `mean`.

```shine
loop i in 0..10 step 0 {
    print(i)
}
```

### Conversion Error

Невозможное преобразование:

```shine
value = number("abc")
```

### File Error

Исходный или пользовательский файл не найден, не читается или не записывается.

### Input Error

Ошибка чтения stdin.

### String Error

Неверное выражение или незакрытая скобка интерполяции:

```shine
print("Value: {missing")
```

### Assertion Error

Ложное условие `assert`:

```shine
assert(2 + 2 == 5, "math is broken")
```

### Control Error

`break` или `continue` вне цикла.

### Runtime Error

Общая защита выполнения, например превышение максимальной глубины рекурсии или лимита условного цикла.

## Рабочий процесс

Рекомендуемый цикл:

```bash
shine fmt src/main.shn
shine check src/main.shn
shine test
shine run src/main.shn
```

Для CI:

```bash
cargo test
shine check examples/program.shn
```

## Как читать ошибку

1. Сначала прочитайте категорию и сообщение.
2. Перейдите к указанной строке и колонке.
3. Посмотрите объяснение после исходной строки.
4. Примените `Suggestion`.
5. Повторите `shine check`.

Не исправляйте только место указателя вслепую: незакрытая скобка или строка выше иногда делает следующую строку проблемной для parser.
